## 项目定位

- 这是个人网站项目，目标是展示真实的人、真实的作品和真实的判断力，不做通用模板站。
- 第一屏必须直接让访问者理解：我是谁、我在做什么、我做过什么，不能只用抽象口号或装饰性视觉占位。
- 默认把 `/home/lgk/projects/AIFocus` 和 `/home/lgk/projects/CodePath` 作为重点项目展示素材来源。
- 网站要服务于长期个人展示，优先清晰、可信、有记忆点，而不是堆功能、堆组件或堆视觉噱头。

## 默认技术与工具策略

- 默认优先使用 Next.js、TypeScript 和 React 搭建，除非后续明确选择其他技术栈。
- 样式、动画和组件方案按项目实际需要选择，优先简单、可维护、可验证。
- 可以主动调用、安装或组合完成任务所需的技能与工具；如果不确定该用什么能力，优先使用可查找 skill 的能力来发现合适工具。
- 涉及 Remotion、视频级动效或叙事型动画时，使用 Remotion 相关 skill 和工具；不要为了普通 UI 微交互动用过重方案。
- 涉及本地页面、截图、响应式和真实交互验收时，优先使用 Browser/Chrome 相关能力打开本地站点并检查。

## 设计与交互原则

- 整体审美要像为本人定制的个人作品现场，不像 SaaS 模板、简历模板或组件库演示页。
- 页面应使用真实内容、真实项目、真实截图或可信的生成视觉资产；不要用空泛图标、假数据和 stock 感占位图撑场面。
- 信息层级要锋利：首屏建立身份和方向，项目区说明价值和证据，细节区展示能力结构与联系方式。
- 动效要克制但有记忆点，服务叙事、空间、状态变化或项目理解；禁止为了炫技牺牲可读性和性能。
- 桌面和移动端都必须是完整体验。移动端不能只是缩小版桌面，也不能出现遮挡、溢出、文字压缩或按钮难点。
- 保持可访问性：语义结构清楚，颜色对比可读，键盘可达，图片有合理替代文本，动画尊重用户的减少动态偏好。
- 避免泛模板感：不要堆叠大量同质卡片，不要使用无意义渐变光斑或装饰形状，不要把说明书式文案直接铺在页面上。

## 开发工作流

- 开始实现前先理解目标和成功标准；遇到多个合理方向时，说明取舍并选择最能满足个人网站目标的方案。
- 默认采用最简单可行方案，外科手术式修改，只触碰完成当前任务必须触碰的文件。
- 新增功能要优先形成可运行、可截图、可评审的完整纵向切片，而不是先铺大量不可见基础设施。
- 使用相邻项目素材前，先确认项目描述、技术栈和当前公开能力，避免写出过期或不准确内容。
- 代码风格跟随当前项目；如果项目还未初始化，建立清晰、少量、可维护的结构，不预设复杂抽象。
- 可以自行制定或调整后续项目管理方式，但要把规则写在合适位置，并优先选择最轻量、最容易执行的流程。
- 发现无关问题可以提醒，但不要顺手修复。不得覆盖用户已有改动。

## 验证要求

- 完成代码或 UI 改动前，必须运行能覆盖本次改动的最小有效验证。
- 对 Next.js/TypeScript 项目，默认至少运行类型检查、lint 或 build 中与当前项目可用脚本对应的命令。
- UI 改动完成后，默认启动本地站点并给出访问地址。
- 交付前必须使用真实浏览器查看页面，至少覆盖桌面和移动端视口，并保存或说明对应截图检查结果。
- 重点检查首屏、项目展示、导航/滚动、动效、移动端布局、可读性、性能直觉和控制台错误。
- 如果某项验证无法执行，必须说明原因、剩余风险和下一步最小补救方式。

## 三 Agent 终审标准

- 成品交付前，必须由 3 个独立 AI subagent 分别评审成品、真实浏览器截图和代码。
- 3 个角色分别是：
  - 重度 vibe coder：评审整体气质、完成度、交互手感、创造性和是否像一个真正有生命力的个人网站。
  - 原生审美设计师：评审视觉系统、版式、节奏、字体、颜色、响应式、细节质感和是否摆脱模板感。
  - 破坏性质量官：评审构建、代码、可访问性、边界情况、移动端、性能、内容真实性和安全红线。
- 每个 subagent 都必须给出 0 到 100 分、红线结论、主要扣分点和可执行修改建议。
- 交付达标条件：三个评分全部大于或等于 90 分，并且没有任何红线。
- 如果任一评分低于 90 或出现红线，必须修复后重新运行相关验证和三 Agent 终审。

## 红线

- 构建失败、类型检查关键失败、运行时主页面报错或本地站点无法打开。
- 移动端出现遮挡、横向溢出、关键文字不可读、按钮不可点或首屏主体缺失。
- 主路径不可用，包括首屏表达不清、项目展示无法理解、关键链接失效或联系方式不可用。
- 展示内容不准确，包括夸大项目能力、伪造数据、错误描述 AIFocus 或 CodePath。
- 泄露 API key、token、密码、私有配置、未公开数据或不应公开的本机路径。
- 页面明显像模板站、组件库拼装页、AI 生成占位页或只靠装饰而没有真实内容。
- 三 Agent 终审任一角色低于 90 分，或任一角色标记红线。
